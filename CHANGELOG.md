## 2026-07-18 - 今日一字与每日题记

- 首页在当天尚未盖章且没有未完成组时展示确定性的「今日一字」，带语境词和拼音；点击直接进入该字练习，首次盖章后自动让位给「今日拾得」。

## 2026-07-18 - 通知即题面与纸墨声音

- iOS 练习提醒从通用召回文案改为「语境词 + 拼音」的单字题面，优先选择到期且上次没写出的字；没有到期或高风险目标时不推送。
- 本地通知携带稳定 `cardKey`，点击后由原生冷/热启动回调进入对应单字练习；当日已盖章、关闭提醒和前台静默口径保持不变。
- 盖章增加低沉短促的原生闷响，描红起笔增加约三分之一感知强度的纸声；两者之外无点击音、结果音或背景音乐，盖章声音、触觉和视觉在同一记账帧触发。
- 「我的」新增默认开启的「纸墨声音」开关并加入备份白名单；关闭后不创建 Web AudioContext。iOS 使用 `AVAudioSession.Category.ambient`，静音键拨下时保持安静。
## 2026-07-18 - 印历补帖、月帖与年度报告

- 「我的」增加练习天数入口；印历按月呈现正常「拾」印、浅色「补」印与留白三态，支持跨月翻阅，减少动态效果时直接显示章印。
- 最近 7 天的空白日可发起 5 字真实补帖；补帖目标随 session v2 恢复，只有完整写完才幂等落印，中途退出不补印，活动记录自动进入备份。

- 「本月拾字帖」生成 1080×1440 本地 PNG，分开呈现当月练过、真正稳定的已拾回、独立写出、练习天数和最难独立字，并复用既有原生 / Web 分享通道；缺少墨迹的字用楷体补位。
- 增加四屏本地年度报告，汇总年度字数、最常提笔月份、最生僻字和第一个字，不上传数据、不做用户比较。
- Web 回归覆盖印历三态与网格几何、跨月、补帖未完成/完成/恢复/幂等、备份、墨迹容量降级、月帖尺寸与分享、年度单屏滚动。

## 2026-07-06 - 自动判对错 v2：连笔不再弃判 + 醒目 ✓/✗ 结论

真机反馈暴露了 v1 的两个致命问题：手机上手指书写连笔是常态，v1 一遇笔画数不符就退到中立文案「你写了 X 笔、标准 Y 笔」弃判——用户几乎每次都撞上这条，等于没有判定；而且结论只藏在一行小字里，没有明确的对/错。

### 判定改造：笔画数不同也照样判
- 笔画数相同：沿用逐笔配对精判（位置/端点/长度，能抓「未/末」）。
- 笔画数不同：新增**整体覆盖度判定**——双向检查「标准的每一笔有没有被墨迹描到」（抓少笔/漏部件）和「用户的每一笔有没有落在标准字形上」（抓多笔/错位）。「描到」要求距离近**且方向一致**（|cos|≥0.5，允许倒笔）：只看距离的话，漏写的竖钩会被几条横「路过」误判为写过了。连笔、断笔对判定完全无影响。
- 校准结果：12 字 × 潦草（每笔随机错位 ±7%）× 连笔，72 例零误判；事漏竖钩、显漏点、口多一撇、味/昧、口/日、未/末、土/士、人/入 全部抓出。
- 已知极限（记录在案，不为它们冒误伤正确书写的风险）：或/戒、日/曰这类墨迹重叠 90%+ 的形近对可能放过——误把写对判成写错比放过形近字更伤信任。

### 结论展示：大 ✓/✗
- 揭晓时在田字格下方给出大字号「✓ 写对了」（绿）/「✗ 写得不太对」（红），不再只有一行小字。
- 判错时默认展示**你的笔迹**（标红直接可见），正确字就在题面和「看正确字」一步之内；判对默认展示正确字，保持原对照习惯。
- 判错文案给出原因：「像是少写了 N 笔」「N 笔没写到位（已标红）」。

### 顺手修复
- 指针事件加 isPrimary 过滤：手机上手掌/第二根手指的触点会把正在写的一笔打断或拆碎。
- `sw.js` 缓存版本 v6→v7。

## 2026-07-05（晚）- 写完自动判对错：本地几何比对，毫秒级反馈

### 新增：「写好了」揭晓时自动比对

不需要手写识别模型：目标字是已知的，判定只需回答「写的和标准像不像」。用户笔迹和字库 medians（笔画中线，`data/*.json` 里本来就有）做纯几何比对——整字包围盒对齐（写小/写偏不吃亏，只允许轻微拉伸避免抹掉字形差异）→ 笔画贪心配对（不苛求笔顺，笔顺练习由「笔顺提示」负责）→ 逐笔卡均距/端点/长度三个阈值。全程本地运算，实测整个揭晓含比对 <15ms，无网络、无模型、无等待。

- 三种结论：写对（✓）；有 N 笔差别较大（这些笔在「看我刚写的」里标红）；笔画数不符（报「你写了 X 笔、标准 Y 笔」，注明可能是连笔或漏笔，不妄下结论）。
- 判定只是「建议」：在自评四档上圈出建议按钮（写对→会写，用过提示→看提示写出，写错→写错了；笔画数不符不圈），最终以用户点选为准——连笔、行草会被几何比对误判，机器不能替人拍板。
- 泄题红线：比对只发生在「写好了」揭晓瞬间，绝不在书写过程中反馈（写一半提示对错=把回忆测试变成试错游戏）；「揭晓答案」（不会写）路径不比对；空白直接点「写好了」也不比对。
- 实测验证：完美书写/写小 18%+抖动 → ✓；「柯」写成「美」（同为 9 笔）→ 9 笔全标红；漏一笔 → 报 8/9；形近对「未」写成「末」（只差两横长短）→ 长度校验抓住。
- 已知局限：连笔行草会被判「笔画数不符」；说不出「你写成了哪个字」（那需要 ML 识别模型，暂不做）。

### 实现
- 笔画记录挂在现有 pointer 事件上，清空、换卡、画布重建时同步重置。
