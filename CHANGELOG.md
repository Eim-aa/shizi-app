# 变更记录

## 2026-07-05（下午）- UX/UI 还原到原始设计稿（首页 + 底部导航）

对照 `汉字书写恢复工具/` 里的设计稿，之前几版把首页从「一个动作的发射台」做成了「控制面板」，底部也从设计的 3 tab 堆成了 5 tab + 内部。本次按设计稿还原：

### 首页：极简 launcher
- 从「今日练习标题 + 出题偏好 pill + 3 个统计方块 + risk 行 + 大按钮 + homeHint + 4 个引导 chip + 错题本/易错分析按钮」减到：问候语 + 一行状态 + **一个大圆按钮** + 底部一行淡统计。
- 大圆按钮承载「今天该练什么」：有到期→「复习 / X 个到期」，否则→「开始 / 15 字」（新用户先校准）。
- 出题偏好移到「我的」设置；引导 chip 移除（新手引导改由状态文案承载）；统计方块移到「我的」。

### 底部导航：5 tab + 内部 → 3 个带图标的 tab
- **练习**（首页 launcher）/ **错题**（错题本）/ **我的**。
- 「我的」聚合：统计卡（已记录/到期/错题，可点直接进复习或错题练习）、易错分析入口、练习设置（出题偏好、加字）、内部工具（题库质检/实验数据/备份/恢复/重置）。
- 练习中底部导航仍收起（专注写字），退出走卡片顶栏「退出本组」。

### 修复
- 首页大圆按钮此前因全局 `button{flex:1}` 在纵向 flex 里被拉成椭圆——加 `flex:0 0 auto` 复位为正圆。
- 首页状态行原会显示整个新字池数量（如「4373 字待写」），改成会话级有意义的文案。
- toast 位置上移，不再压在底部 tab 栏上。
- `verify_8105_app.js` 适配新 DOM（3 tab、我的页统计、大按钮、渲染入口）。

## 2026-07-05 - 修复答案泄漏与练习专注度回归 + 题库数据独立

### P0 修复：题面泄漏答案

上一版给练习卡加的语境说明句（「词语：起来。请根据这个常见语境，写出拼音 qǐ 对应的缺字。」）把**含目标字的完整词语直接印在了题面上**——每张卡的答案都写在题目里，回忆测试完全失效。已删除该句：题面的挖空词＋语境字＋拼音本身就是「词语＋语境＋拼音」，不需要（也不能）用文字复述。

### 练习专注度修复

- 「笔顺提示」从最大的主按钮降回辅助按钮，主按钮还给「写好了」——产品测的是回忆，提示是兜底不是入口。
- 修复：用过提示后用户笔迹永久卡在 22% 透明度。现在播放中淡化、播完自动恢复。
- 取消「全部提示播完强制进入自评」：看完笔顺通常还要照着写，何时自评由用户决定。
- 练习中收起底部 Tab（写字时三层按钮叠在底部易被手掌误触）；「退出本组」「＋加字」收进卡片顶栏，练习中加字插队功能保留。
- 自评 toast 从每张必弹降为只在字首次进入错题本时提示；质量反馈、整组反馈面板内已有反馈的不再重复弹 toast；自评区长说明精简回一句话。

### 调度修复

- 「写错了」的复习间隔此前恒为 0.5 天（depth 取了已归零的 streak）；现按累计次数递进（0.5 → 1 → 3 天），与「不会写」「看提示写出」一致。

### 架构

- 内嵌的 1.9MB 题库（SEED/GROUPS）抽离为独立的 `deck-data.js`：index.html 从 2MB 降到约 90KB，界面迭代不再让用户重新下载整个题库；题库更新时 bump `sw.js` 的 VERSION 即可整体换新。`scripts/build_8105_chars.py` 已同步改为输出 `deck-data.js`。

### 保留的上一版改进

错题本/易错分析命名、四档自评新语义、简单/中等/难用户侧难度、底部 Tab、结果页薄弱优先排序、退出本组、质量反馈可撤销、练习中加字插队、首页新手引导——方向都对，全部保留。

## 2026-07-04 - UX 回归问题修复

对应提交：`1ebcd54 Fix UX regression feedback flow`

这次不是大改版，而是集中修复测试反馈里的 P0/P1 产品体验问题。目标是让当前 MVP 更容易开始、更不容易迷路，并且让用户每次操作后都能立刻感知系统响应。

### 导航与新手引导

- 首页新增轻量新手引导：
  - 看词语和语境
  - 在田字格写
  - 用笔顺提示
  - 自评后复习
- 恢复稳定的底部 Tab 导航：
  - 首页
  - 练习
  - 错题
  - 分析
  - 加字
  - 内部
- 底部导航行为优化：
  - 点“练习”时，若有到期复习，优先进入复习；否则开始新一组。
  - 离开内部页面时自动收起内部工具菜单，避免底部导航混乱。
- 用户侧命名从“重点字本”统一改成“错题本”，更符合用户预期。
- 练习页、结果页、错题本、分析页等非首页页面都保留明确返回路径。

### 练习卡

- 出题方式从“只看拼音”改成“词语 + 语境 + 拼音”：
  - 练习说明改成“根据词语和语境写出缺字”。
  - 卡片会给出类似“词语：啰嗦。请根据这个常见语境，写出拼音 luō 对应的缺字。”的提示。
- 自评选项统一为：
  - 会写
  - 看提示写出
  - 不会写
  - 写错了
- “笔顺提示”变成更大的主按钮。
- “揭晓答案”降为次级操作。
- 点击笔顺提示时，会先淡化用户笔迹，避免提示笔画和用户书写重叠。
- 当前字的全部笔顺提示展示完后，自动进入自评状态，不再要求用户额外点击“写好了”。
- 自评区增加轻量预提示，让用户知道不同按钮会影响后续复习。
- 自评后增加 toast 反馈，例如“已进错题本”“后面会少复习”等。

### 反馈与纠错

- “不会写”“写错了”“看提示写出”都会被视为薄弱信号，进入复习和错题逻辑。
- “写错了”不再按接近掌握处理，而是安排更近的复习。
- 题目质量反馈改成可撤销：
  - 太简单
  - 太生僻
  - 词不自然
  - 不想练
- “不想练”不再立即跳过当前题，避免误触导致数据污染和流程中断。
- 整组难度反馈增加即时 toast 确认。

### 难度模型与展示

- 用户侧难度展示从学历标签改成能力标签：
  - 简单
  - 中等
  - 难
- 生成数据里仍保留内部五级标签用于抽题，但产品界面不再展示“初中/高中/大学/专业”这类学历导向标签。
- 一组内的自适应逻辑保留并继续生效：
  - 连续“会写”会提高后续题目目标难度。
  - “看提示写出”和“写错了”会小幅降低目标难度。
  - 连续“不会写”会更明显降低目标难度。

### 结果页

- 结果页按薄弱项优先排序：
  - 不会写
  - 看提示写出
  - 写错了
  - 会写
- “马上复习错题”上移到结果列表上方，缩短复习路径。
- 增加明确的“返回首页”按钮。
- 结果页里的单字仍然可点击，能直接进入专项练习。

### 加字流程

- 练习中手动加字后，新字会立即插入当前训练队列，从下一题开始出现。
- 加字成功后会显示 toast，提示“已加入本次练习”。
- 原有逻辑保留：
  - 字库内的字使用真实卡片数据。
  - 字库外的字生成自定义卡片。
  - 加入的字会按一次现实中的“不会写”记录进记忆模型。
