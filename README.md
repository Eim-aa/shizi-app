# 拾字 PWA Beta

这是一个本地运行、可安装到手机主屏幕的汉字书写恢复工具。当前目标是 Pre-App 阶段的高保真功能合流版：先把真实记忆逻辑和高保真核心体验打磨成可连续自测的 PWA Beta，再决定是否进入原生 App / TestFlight / App Store。

## 主要文件

- `index.html`：产品主页面（界面与逻辑，约 120KB）。
- `deck-data.js`：内置题库数据（SEED 词卡 + GROUPS 部件分组，约 1.9MB，由生成脚本产出，勿手改）。
- `hanzi-writer.min.js`：汉字笔顺渲染库。
- `fsrs6.min.js`：固定版本、完全离线的 FSRS-6 跨天调度器。
- `THIRD_PARTY_NOTICES.md`：第三方库版本与许可证。
- `data/`：Hanzi Writer 的单字笔画数据。
- `scripts/`：生成数据和验证页面的脚本。
- `generated/`：当前生成出的 8105 字覆盖率报告和 6854 个可写候选字结构化结果。
- `sources/`：生成脚本依赖的原始字表、字频和词典数据。
- `ios/ShiziApp/`：iOS 原生 WKWebView 外壳工程，用于真机安装和 TestFlight 内测。
- `CHANGELOG.md`：按版本记录产品逻辑和 UX 调整。
- `requirements.txt`：生成脚本需要的 Python 依赖。

当前界面已经接入设计师新版高保真方向：整体视觉从工具面板改成「宣纸 × 朱砂印」的产品语言，首启页直接讲清楚“提笔忘字”的场景，首页以“拾”印章作为主入口，并在下方保留一个轻量的即时加字入口。核心数据仍来自真实字库、记忆模型、复习调度和本地存储，界面改版不改变题库和记忆模型的底层逻辑。

练习卡的题面是「词语挖空＋拼音」：其他字给出语境，缺的字只显示拼音，避免同音字歧义，也不会把答案泄在题面上。点「写好了」后进入「提交字格 / 标准字」对照；提交字格冻结当时仍可见的提示笔画和用户墨迹，结果显示与自动助手读取同一份完整快照。不会写时点「不会写」，先照轮廓描一遍，再隐藏答案立即独立写一次；描写只算教学，不算掌握。

结果页提供「写对了 / 写错了」两个明确动作，以及小号的「拿不准，记不清」出口；记不清会盖「补拾」、进入本组再练，不会直接毕业。当逐笔几何检查明确认为差异较大、且本回合没用提示时，选择写对会出现一次轻量确认，整体覆盖度检查不拦截，最终仍由用户决定。系统根据本回合事实自动归类为会写 / 看提示写出 / 写错了 / 不会写，并保留拾到 / 补拾 / 待补 / 再拾的盖章动画和历史标签。普通反馈最多停留约 1.4 秒，点击结果区可立即换题，连续点击不会跨过下一题；「重盖」会原子回滚记忆、组内队列、每日统计和对应 FSRS 事件，再回到同一份提交快照。

普通组达到 20 次尝试或约 6 分钟后，只在换题间隙提醒一次。可以继续完成本组，也可以把尚未写稳和还没写到的字按原顺序排到明天最前面；选择继续后，本组不再重复提醒。首次校准仍完整走完摸底和结果说明，不被节奏提醒截断。

「笔顺提示」是普通回忆阶段的辅助按钮：播放时淡化用户笔迹避免重叠，播完自动恢复；撤销按笔画/提示组逐步回退，重写会清空墨迹与提示画面，但本回合曾使用提示的历史不会归零。看提示写对、自由书写写错和点击不会写都会在本组稍后再写；只有一次全新的无提示、无描写独立写对才能毕业。「按住看一眼 · 不计」会暂时显示已加载的完整真实字形，松手即隐藏，不改变提示次数或本回合归类；原双指透视仍保留并遵守同样的落墨保护。关键控件点击区不低于 44pt，矮屏仍保留引导文字并将四个书写工具收成一行。练习顶栏使用标准返回符号，点击返回和 iOS 左缘侧滑都会先保存一致快照，再直接回首页；保存失败则留在当前页提示重试。

点「写好了」的瞬间会做本地自动比对（毫秒级、离线）：笔画数相同走逐笔配对，笔画数不同或提示与用户笔迹重叠时走整体覆盖判断，连笔、断笔和重复覆盖不会机械地被当成多余笔画。对照助手只高亮建议，不预提交、不锁定选择，最终正确性始终以用户判断为准；比对不会在书写过程中泄题。

当前练习节奏：系统计划通常每组 15 个不同目标字，到期复习或题库末尾可能形成短组；每天最多安排 30 个到期字，按逾期与近期困难程度优先，练完后即可回到新字。练习中主动加字可让本组增长到 16、17……，不挤掉原计划字。同一困难字在组内重复任意次数仍只占一个目标名额。顶部只显示「已练 X/Y」和按字去重的「还要再练 N」；所有目标字出现但仍有困难字时不会提前结算。session v2 会保存墨迹、提示层、提交快照、描写阶段、再练队列和手动下一题队列；返回、刷新、跨午夜或 iOS 杀后台后都可原位继续。当天完整做完第一组后，「今日已拾」完成态不再回退；首页同时显示本月实际练过的天数，不设月目标。累计第 1 / 7 / 14 / 30 个练习日，以及第 100 天后每满 100 天会显示一次里程碑；历史继承跳过的档位不会补弹庆祝。

跨天调度使用固定的 `ts-fsrs 5.2.0 / FSRS-6.0`：期望保留率 0.90、最长间隔 365 天、开启 interval fuzz，只使用 Again 与 Good。看提示写出、写错和不会写记录 Again；最终无辅助独立写对记录 Good。组内重现完全由再练队列控制，不采用 FSRS 的分钟/小时 due；困难字毕业后才启用按本地自然日记录的 `dueDay`，且最早是次日。

「卡点分析」根据本地练习记录汇总用户容易卡住的主题、能力难度和字形结构，只负责解释模式，不再重复展示一套待拾回单字。点任一洞察会进入字盒的待拾回筛选，由字盒统一承接查看和专项练习；从结算页进入时，返回仍回原结算页。

首页优先横滑展示全部「今日拾得」（最多 50 字），没有今日记录时回退到昨日或上回；字盒默认突出最多两行待拾回，完整字墙折叠在「全部 N 字」中。首页与结算只展示当日复习预算，不显示积压总债务；全量到期数在字盒和开发详情中查看。FSRS 只负责毕业后的日期，不接管组内出题顺序。

新用户第一次点击「先练 15 字」会进入一组轻校准题，首页会提示“先用 15 个字估一下难度”。前两张固定为熟悉但容易提笔卡住的「尴 / 嚏」，尽早让用户感受到产品用途；中段完成难度摸底，最后四张封顶在 85，避免最难题紧贴结算。首卡的「笔顺提示 / 不会写」从渲染完成起即可使用，3 秒只切换帮助文案、不锁按钮；校准期第一次卡住会就地说明忘了很正常。已有练习记录的用户不会被重新打断。

每组 15 字内部会做会话内自适应：连续“会写”会把后续新字推向更高难度，“看提示写出”“记不清”和“写错了”小幅回退，连续“不会写”才明显降级。首次校准只有在用户自评写对与几何检查一致时才完整计入挑战档门槛；几何不一致或不可用的写对按半次计，避免早期难度被误推过高。每组结束后还可以标记“太简单、刚刚好、太难、词不自然”，这些整组反馈会持久化为个人难度偏移和语境严格度，下次打开仍然生效。

产品不再承诺固定“六分钟”：首启和首页统一说明“一组字，按自己的节奏”，因为描写与组内再练会按实际失手次数自然增长。校准结算页在 iOS 上就地邀请开启练习提醒，默认仍关闭；浏览器则直接显示明日日期，给出明确的回访预期。

首页可以切换出题偏好：高频易忘、偏实用、偏挑战。每张题卡也可以标记题目质量：太简单、太生僻、词不自然、不想练。这些反馈不会覆盖记忆反馈，而是用于调整后续候选池和难度排序。题目质量反馈是开关，点一次记录、再点一次取消；「不想练」不会立即跳题，避免误触造成数据偏差。

底部导航是 3 个 tab：**练习 / 字盒 / 我的**。
- **练习**：首页，一个极简 launcher——问候语 + 大印章「拾」+ 轻量即时加字 + 最近拾得。新用户先进入 15 字校准；老用户优先完成每天最多 30 个到期字，预算耗尽后可继续练新字。
- **字盒**：待拾回字的权威主场；头部先显示练过、已拾回和最近拾得，再展示待拾回、到期复习与可筛选完整字墙，单字可点击专项练；图例可打开四章词典。
- **我的**：统一显示「练过 / 已拾回 / 待拾回」三个指标；已拾回只计算真正稳定的字，待拾回与它不重叠。练习设置和提醒位于主层级，备份、恢复和清空记录收进“数据管理”。

「我的 → 更大字体」提供标准/大字两档。字号偏好保存在本机并进入备份，所有主要文字通过同一个缩放变量同步变化；承载状态、图例和动作后果的文字至少使用 13px 与较高对比色。320×568 矮屏下练习动作仍在首屏，揭晓页保留卡点建议，较低优先级的题目反馈可在揭晓区域内滚动到达。

「加字」用来抓现实里提笔忘的字：应用内输入面板边输边预览识别出的字（在库字直接带出拼音，练过的会标出来），Enter 或「加入卡组」确认。入口在首页大印章下方、「我的」、字盒和练习卡顶栏（＋加字）。输入的字若在基础字库中，会直接激活对应卡片（带拼音、语境词和部件提示），并按一次真实的「不会写」记入记忆模型，之前的「太简单/太生僻」等排除标记也会被清掉；练习中加入会在当前完整回合结束后按 FIFO 连续出现，优先于已就绪再练字，满 15 字也可增长；本组已有字只提前或复写，不增加分母，末题反馈期加入后退出也会在恢复时先完成复写。练习外加入会按添加顺序排在下一组非校准练习最前，写过一次后恢复正常调度。校准组中加入的额外字同样下一题出现并正常参与学习，但不替换原 15 个摸底样本，也不进入校准统计和调参。不在字库中的字才会生成自定义卡片。重置卡组时自定义字一并清空。

「我的」里的数据管理收着导出备份、恢复备份和清空记录，并常驻显示上次成功备份时间；完成 5 个练习日或记录 30 字后，连续 7 天未备份会温和提醒，第 3 个完成日的结算页也会提示一次。备份按白名单导出用户数据，并包含 session v2、FSRS 状态与日志、描写教学标记和本地漏斗，不包含测试、瞬态键或设备内安全副本。漏斗只在本机幂等记录首启、校准、次日回访、对照分歧、备份和完整组时长，不含设备标识，也不会主动联网发送。恢复前会对比当前/备份字数和最后日期；恢复或清空前自动保留最近一份设备内安全副本，操作后可撤销一次。localStorage 仍是唯一存储，安全副本不能抵御卸载或整站数据被清除，换手机或清浏览器数据前仍需导出备份。

每组结束后会进入结果页：用「今日拾得」字帖展示本组字，并用颜色、形状和「拾/补/待/再」小角标共同标出这次盖的章；每个字块还常驻显示「会写 / 看提示 / 写错 / 没写出」，不要求用户猜颜色或章名。有困难结果时会出现「马上再拾」口袋和「看看你卡在哪」次级入口；口袋只收本组薄弱字，可直接开始专项练习，卡点分析返回时仍回结果页。结果字帖可生成本地 PNG：iOS 调起系统分享面板，浏览器优先使用 Web Share，不支持时下载图片。图片只包含本组字、结果、日期和本次里程碑，不读取用户、设备或完整练习记录。待拾回单字仍由字盒统一管理，结果字帖中的单字也可直接专项练习。

PWA 已配置 manifest、iOS 主屏幕图标、standalone 显示模式和 service worker。安装时会预缓存首日校准组与高频核心共 600 字（约 1.23 MiB），应用外壳、FSRS-6 和这些笔顺可直接离线运行；其余字首次需要联网下载一次，随后按需缓存。离线未缓存时页面会明确说明原因，仍允许提交并自行判断，「不会写」教学使用系统字形 fallback，不会卡死在单张卡片；恢复网络后未落笔的当前卡会自动重试。

## 线上版本（发给朋友测就用这个）

- 地址：<https://eim-aa.github.io/shizi-app/>
- 发布方式：GitHub Pages 从 `main` 分支根目录自动构建——`git push` 到 main 就是发布，一两分钟后生效。页面走「网络优先」缓存策略，用户下次打开会自动拿到新版。
- 手机安装：iPhone 上用 Safari 打开 → 分享按钮 → 「添加到主屏幕」，之后从主屏幕图标进入即全屏、可离线。
- 提醒测试的朋友：练一阵后去「我的 → 数据管理 → 导出备份」发回 JSON。每位测试者只保留最新一份，然后用 `python3 scripts/summarize_backups.py 备份目录/` 汇总 D1/D7、校准完成率、系统分歧率和平均组时长。

## 本地预览

在这个目录中运行：

```bash
python3 -m http.server 8000
```

然后打开：

```text
http://127.0.0.1:8000/
```

开发调试时可以打开：

```text
http://127.0.0.1:8000/?dev=1
```

## iOS App / TestFlight

第一阶段 iOS 工程在 `ios/ShiziApp/ShiziApp.xcodeproj`，用 WKWebView 加载现有静态前端，不改写业务逻辑。每次 Xcode Build 会把 `index.html`、题库、`hanzi-writer.min.js`、`fsrs6.min.js` 和整个 `data/` 笔画目录同步进 App 包；普通 scheme 不显示开发工具，`Shizi Dev` scheme 会等价打开 `?dev=1`。

构建、真机运行、Safari Web Inspector 和 TestFlight 上传步骤见 `ios/ShiziApp/README.md`。

## 重新生成题库

第一次运行前安装依赖：

```bash
python3 -m pip install -r requirements.txt
```

重新生成 `index.html` 内置题库、`generated/selected_8105_candidates.json` 和 `generated/hanzi_writer_coverage.json`：

```bash
python3 scripts/build_8105_chars.py
```

运行浏览器验证：

```bash
node scripts/verify_8105_app.js
```

汇总测试者主动回传的备份（目录中每人只放最新一份；加 `--json` 可输出机器可读结果）：

```bash
python3 scripts/summarize_backups.py /path/to/backups/
```
