# 拾字 PWA Beta

这是一个本地运行、可安装到手机主屏幕的汉字书写恢复工具。当前目标是 Pre-App 阶段的高保真功能合流版：先把真实记忆逻辑和高保真核心体验打磨成可连续自测的 PWA Beta，再决定是否进入原生 App / TestFlight / App Store。

## 主要文件

- `index.html`：产品主页面（界面与逻辑，约 120KB）。
- `deck-data.js`：内置题库数据（SEED 词卡 + GROUPS 部件分组，约 1.9MB，由生成脚本产出，勿手改）。
- `hanzi-writer.min.js`：汉字笔顺渲染库。
- `fsrs6.min.js`：固定版本、完全离线的 FSRS-6 跨天调度器。
- `THIRD_PARTY_NOTICES.md`：第三方库版本与许可证。
- `data/`：Hanzi Writer 的单字笔画数据。
- `scripts/`：生成数据和验证页面的脚本。
- `generated/`：当前生成出的 8105 字覆盖率报告和 6854 个可写候选字结构化结果。
- `sources/`：生成脚本依赖的原始字表、字频和词典数据。
- `ios/ShiziApp/`：iOS 原生 WKWebView 外壳工程，用于真机安装和 TestFlight 内测。
- `CHANGELOG.md`：按版本记录产品逻辑和 UX 调整。
- `requirements.txt`：生成脚本需要的 Python 依赖。

当前界面已经接入设计师新版高保真方向：整体视觉从工具面板改成「宣纸 × 朱砂印」的产品语言，首启页直接讲清楚“提笔忘字”的场景，首页只保留一个“拾”印章作为开始入口。核心数据仍来自真实字库、记忆模型、复习调度和本地存储，界面改版不改变题库和记忆模型的底层逻辑。

练习卡的题面是「词语挖空＋拼音」：其他字给出语境，缺的字只显示拼音，避免同音字歧义，也不会把答案泄在题面上。点「写好了」后进入「提交字格 / 标准字」对照；提交字格冻结当时仍可见的提示笔画和用户墨迹，结果显示与自动助手读取同一份完整快照。不会写时点「不会写」，先照轮廓描一遍，再隐藏答案立即独立写一次；描写只算教学，不算掌握。

结果页只让用户判断「写对了 / 写错了」。系统根据本回合事实自动归类为会写 / 看提示写出 / 写错了 / 不会写，并保留拾到 / 补拾 / 差点 / 回炉的盖章动画和历史标签。普通反馈最多停留约 1.4 秒，点击结果区可立即换题，连续点击不会跨过下一题；「改一下」会原子回滚记忆、组内队列、每日统计和对应 FSRS 事件，再回到同一份提交快照。

普通组达到 20 次尝试或约 6 分钟后，只在换题间隙提醒一次。可以继续完成本组，也可以把尚未写稳和还没写到的字按原顺序排到明天最前面；选择继续后，本组不再重复提醒。首次校准仍完整走完摸底和结果说明，不被节奏提醒截断。

「笔顺提示」是普通回忆阶段的辅助按钮：播放时淡化用户笔迹避免重叠，播完自动恢复；撤销按笔画/提示组逐步回退，重写会清空墨迹与提示画面，但本回合曾使用提示的历史不会归零。看提示写对、自由书写写错和点击不会写都会进入本组巩固；只有稍后一次全新的无提示、无描写独立写对才能毕业。首次用过提示后会说明“双指按住格子，可以透视提示”；透视会取消尚未抬笔的半笔，期间不落墨，松开任一指即恢复。练习顶栏使用标准返回符号，点击返回和 iOS 左缘侧滑都会先保存一致快照，再直接回首页；保存失败则留在当前页提示重试。

点「写好了」的瞬间会做本地自动比对（毫秒级、离线）：笔画数相同走逐笔配对，笔画数不同或提示与用户笔迹重叠时走整体覆盖判断，连笔、断笔和重复覆盖不会机械地被当成多余笔画。对照助手只高亮建议，不预提交、不锁定选择，最终正确性始终以用户判断为准；比对不会在书写过程中泄题。

当前练习节奏：系统计划通常每组 15 个不同目标字，到期复习或题库末尾可能形成短组；练习中主动加字可让本组增长到 16、17……，不挤掉原计划字。同一困难字在组内重复任意次数仍只占一个目标名额。顶部只显示「已练 X/Y」和按字去重的「待巩固 N」；所有目标字出现但仍有困难字时不会提前结算。session v2 会保存墨迹、提示层、提交快照、教学阶段、巩固队列和手动下一题队列；返回、刷新、跨午夜或 iOS 杀后台后都可原位继续。当天完整做完第一组后，「今日已拾」完成态不再回退；累计第 1 / 7 / 30 / 100 个练习日会显示里程碑。

跨天调度使用固定的 `ts-fsrs 5.2.0 / FSRS-6.0`：期望保留率 0.90、最长间隔 150 天，只使用 Again 与 Good。看提示写出、写错和不会写记录 Again；最终无辅助独立写对记录 Good。组内重现完全由巩固队列控制，不采用 FSRS 的分钟/小时 due；困难字毕业后才启用按本地自然日记录的 `dueDay`，且最早是次日。

“手感诊断”会根据本地练习记录，先给出“今天先补什么”的行动建议，再汇总用户容易卡住的主题、能力难度、字形结构和具体错题。里面的主题、难度、结构和单字都可以点击，直接进入专项练习。字第一次进入错题本时会有 toast 提示，让「标记看提示写出/不会写/写错了 → 自动进错题本」这条链路可感知。

首页优先横滑展示全部「今日拾得」（最多 50 字），没有今日记录时回退到昨日或上回；字盒默认突出最多两行待拾回，完整字墙折叠在「全部 N 字」中。跨天到期字继续按既有计划进入每日组，FSRS 只负责毕业后的日期，不接管组内出题顺序。

新用户第一次点击「先练 15 字」会进入一组轻校准题，首页会提示“先用 15 个字估一下难度”。题目按中等到高难阶梯展开，并根据本组反馈自动设置初始出题偏好和个人难度偏移。已有练习记录的用户不会被重新打断。

每组 15 字内部会做会话内自适应：连续“会写”会把后续新字推向更高难度，“看提示写出”和“写错了”小幅回退，连续“不会写”才明显降级。每组结束后还可以标记“太简单、刚刚好、太难、词不自然”，这些整组反馈会持久化为个人难度偏移和语境严格度，下次打开仍然生效。

首页可以切换出题偏好：高频易忘、偏实用、偏挑战。每张题卡也可以标记题目质量：太简单、太生僻、词不自然、不想练。这些反馈不会覆盖记忆反馈，而是用于调整后续候选池和难度排序。题目质量反馈是开关，点一次记录、再点一次取消；「不想练」不会立即跳题，避免误触造成数据偏差。

底部导航是 3 个 tab：**练习 / 字盒 / 我的**。
- **练习**：首页，一个极简 launcher——问候语 + 一个大印章「拾」+ 昨日拾得。新用户先进入 15 字校准；老用户有到期先复习、否则练新字。
- **字盒**：先展示最需要处理的待拾回字；全部字默认折叠，展开后可按全部、待拾回、已稳筛选，到期字保留复习入口，单字可点击专项练。
- **我的**：统计（已拾回 / 练习天数 / 回炉中）、手感诊断入口、练习设置（出题偏好、加字）、练习提醒都放在主层级；备份、恢复和清空记录收进“数据管理”，避免普通用户被开发/运营工具打断。

「加字」用来抓现实里提笔忘的字：应用内输入面板边输边预览识别出的字（在库字直接带出拼音，练过的会标出来），Enter 或「加入卡组」确认。入口在「我的」和练习卡顶栏（＋加字）。输入的字若在基础字库中，会直接激活对应卡片（带拼音、语境词和部件提示），并按一次真实的「不会写」记入记忆模型，之前的「太简单/太生僻」等排除标记也会被清掉；练习中加入会在当前完整回合结束后按 FIFO 连续出现，优先于已就绪巩固字，满 15 字也可增长；本组已有字只提前或复写，不增加分母，末题反馈期加入后退出也会在恢复时先完成复写。练习外加入会按添加顺序排在下一组非校准练习最前，写过一次后恢复正常调度。校准组中加入的额外字同样下一题出现并正常参与学习，但不替换原 15 个摸底样本，也不进入校准统计和调参。不在字库中的字才会生成自定义卡片。重置卡组时自定义字一并清空。

「我的」里的数据管理收着导出备份、恢复备份和清空记录，并常驻显示上次成功备份时间；记录超过 100 字且 14 天未备份时会温和提醒。备份按白名单导出用户数据，并包含 session v2、FSRS 状态与日志、描写教学标记，不包含测试/瞬态键；恢复时校验格式并确认后整体覆盖。localStorage 是唯一存储，换手机或清浏览器数据前先备份。

每组结束后会进入结果页：用「今日拾得」字帖展示本组字，并用颜色角标标出这次盖的章。如果卡住较多，结果页会把待拾回字上移成「马上再拾」入口，避免用户继续刷新题；单字同样可以点击，直接进入专项练习。

PWA 已配置 manifest、iOS 主屏幕图标、standalone 显示模式和 service worker。应用外壳、FSRS-6 和已缓存笔顺可离线运行；若某字没有笔顺数据，页面仍允许提交并自行判断，「不会写」教学使用系统字形 fallback，不会卡死在单张卡片。

## 线上版本（发给朋友测就用这个）

- 地址：<https://eim-aa.github.io/shizi-app/>
- 发布方式：GitHub Pages 从 `main` 分支根目录自动构建——`git push` 到 main 就是发布，一两分钟后生效。页面走「网络优先」缓存策略，用户下次打开会自动拿到新版。
- 手机安装：iPhone 上用 Safari 打开 → 分享按钮 → 「添加到主屏幕」，之后从主屏幕图标进入即全屏、可离线。
- 提醒测试的朋友：练一阵后去「内部 → 备份数据」导出 JSON 发回来，里面有打开天数、每组反馈和练习记录，就是留存实验的原始数据。

## 本地预览

在这个目录中运行：

```bash
python3 -m http.server 8000
```

然后打开：

```text
http://127.0.0.1:8000/
```

开发调试时可以打开：

```text
http://127.0.0.1:8000/?dev=1
```

## iOS App / TestFlight

第一阶段 iOS 工程在 `ios/ShiziApp/ShiziApp.xcodeproj`，用 WKWebView 加载现有静态前端，不改写业务逻辑。每次 Xcode Build 会把 `index.html`、题库、`hanzi-writer.min.js`、`fsrs6.min.js` 和整个 `data/` 笔画目录同步进 App 包；普通 scheme 不显示开发工具，`Shizi Dev` scheme 会等价打开 `?dev=1`。

构建、真机运行、Safari Web Inspector 和 TestFlight 上传步骤见 `ios/ShiziApp/README.md`。

## 重新生成题库

第一次运行前安装依赖：

```bash
python3 -m pip install -r requirements.txt
```

重新生成 `index.html` 内置题库、`generated/selected_8105_candidates.json` 和 `generated/hanzi_writer_coverage.json`：

```bash
python3 scripts/build_8105_chars.py
```

运行浏览器验证：

```bash
node scripts/verify_8105_app.js
```
