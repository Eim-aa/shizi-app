这份清单用于第一阶段 WKWebView 封装验收。每次 TestFlight 前至少跑一台刘海屏/灵动岛 iPhone；如果有旧机，再补一台小屏设备。

## 自动真机 Smoke

- [ ] `signing-preflight.sh` 通过，设备状态为 available。
- [ ] 运行 `DEVICE_ID='设备名称或 UUID' SIGNING_XCCONFIG=ios/ShiziApp/Config/Signing.local.xcconfig ios/ShiziApp/scripts/smoke-device.sh`。
- [ ] 普通模式两次 native smoke 均通过，第二次确认 `localStorage` 跨进程重启保留。
- [ ] Smoke 的 `handwritingFlow` 全部通过，确认 pointer 轨迹能落墨且触摸滚动被阻止。
- [ ] Smoke 的软件键盘严格检查通过，`keyboardInsetActive` 为 true，输入框和确认按钮均可达。
- [ ] 加 `DEV_MODE=1` 再跑一次，确认题库质检入口可见且流程通过。
- [ ] 将 `ios/ShiziApp/build/device-smoke/` 下的 JSON 结果随验收记录保留。

## 手写练习

- [ ] 进入写字页后田字格尺寸正常，没有被安全区或底部区域遮挡。
- [ ] 小屏设备上「撤销 / 重写 / 看答案 / 写好了」全部在首屏内，页面不可滚动。
- [ ] 手指在田字格内书写流畅，笔迹不断裂、延迟可接受。
- [ ] 手指写字时页面不滚动、不回弹。
- [ ] 「撤销」按动作倒序回退：写一笔、看一组提示、再写一笔后，连续点三次依次撤回笔画、提示组、笔画；提示组回退瞬时重绘且不闪烁。
- [ ] 普通练习点「重写」会把墨迹和已显示提示全部清空，提示计数回到 0/M；已经用过提示的账不回退，主按钮仍为「写完了」，重看同一组提示不重复记账。
- [ ] 落笔中及抬笔后约 300ms，「看答案 / 写好了」不会被手掌误触。
- [ ] 「笔顺提示」可以播放，播放后仍能继续书写。
- [ ] 双指或意外三指进入透视后，任一指抬起/取消都立即恢复；余下手指移动不续写，全部离开后下一次落笔正常。
- [ ] 动作标签按提示状态流转：未提示为「看答案 / 写好了」，用过提示为「看答案 / 写完了」；提示全部用完后隐藏「看答案」，只留「写完了」，并提示直接照着格中整字描一遍。
- [ ] 提示用完后直接在格中描完并点「写完了」，对照页建议「补拾」；撤销最后一组提示或重写后「看答案」恢复，态 C 引导撤下。
- [ ] 「看答案」留在原田字格显示淡红轮廓；未描时「补拾」不可点，落笔后可点。
- [ ] 描红页放弃按钮显示「不会写」；说明里明确「描一遍也算拾回」和不想描时可照实选「不会写」。
- [ ] 描红后仍可自选「回炉」，且下一次出现时间按所选印章计算。
- [ ] 用过一次笔顺提示后只出现一次「双指按住格子，可以透视提示」引导；双指按住时墨迹淡出、提示增强，松开任一指恢复。
- [ ] 写到一半再放下第二指，未完成的半笔被取消；透视期间移动任一手指都不会新增墨迹；描红和揭晓态不会触发透视。
- [ ] 双指进入和退出透视时，「看答案 / 写好了」区域透明度稳定，不会闪暗一下。
- [ ] 「写好了」可以进入对照页。
- [ ] 四个章始终以「会写 / 看提示写出 / 写错了 / 不会写」为主标签。
- [ ] 除校准组第一章外，盖章结果停留约 1.1 秒后自动进入下一题，没有「下一题」按钮；停留期点击结果区可立即跳过，快速连点不会多跳一题。
- [ ] 校准组第一章停留一次解释印章影响，随后自动进入下一题。
- [ ] 系统判定为接近正确时可显示正向建议章圈；判定为不匹配时不标红笔迹、不预选负向章，只提示与标准字对照并由用户判断。
- [ ] 下一题顶部 5 秒内显示「改一下」；点击后回到上一题揭晓态，下一题仍未作答。
- [ ] 「改一下」为浮层，出现/消失时田字格、题面和按钮位置不跳动；浮层只占头部行和顶部留白，默认屏/小屏都不遮题目，开始写下一题后立即隐藏。
- [ ] 「改一下」会同步撤回记忆、调度、今日章数和进度点，不留下重复记录。
- [ ] 顶部退出本组路径合理：已自评保留，当前未自评卡不记录。

## 本地数据

- [ ] 完成至少一题后退出并重启 App，已拾字/回炉/复习状态仍存在。
- [ ] 未完成 15 题时退出，首页显示「今天拾了 N 个 · 继续这组」；重启后从同一题继续。
- [ ] 完成第一组后首页固定显示「今日已拾 N 个字」；到期不足 15 字的短组做完也算完成，加练或新增到期字不会把完成态改回未完成。
- [ ] 快速连点盖章、重写等控件不会触发页面双击缩放，双指捏合缩放仍可用。
- [ ] 完成后加练一组，今日计数继续累加；中途退出显示「继续加练这组」。
- [ ] 首页「拾字第 N 天」按累计练习日显示：当天完整做完第一组才 +1，只盖几章或只打开不计；与「我的」页「练习天数」一致，且同日加练不重复计。
- [ ] 老数据升级后，累计天数 = 升级前的历史打开天数 + 升级后的完成日数，不清零。
- [ ] 首个完成日结算页出现「见字如晤」庆祝行，且只出现这一次。
- [ ] 校准轮有未写出的字时，结果页出现「先别走」卡；点「马上再拾」进入本轮薄弱字重练。
- [ ] 同一天先盖「不会写」、后在重练中拾回的字，次日仍会进入待复习列表一次。
- [ ] 练习中「＋加字」从下一题出现；练习外连续添加两个字，下一组按添加顺序排在最前，写过后不再强制插队。
- [ ] 校准组中新增字不插队；完成校准后的下一组排在最前。首页加字后的提示能区分这两种情况。
- [ ] 首页有今日记录时显示「今日拾得」并包含四类结果；超过 4 个字可顺滑横滑浏览，左右边缘渐隐且后续字自然半露；无今日记录时回退「昨日拾得/上回拾得」，不会写的字为克制灰态。
- [ ] 字盒打开后先看到统计和最多两行「待拾回」，「全部 N 字」默认折叠，「＋加字」在折叠态仍可见；待拾回为空时显示「没有待拾回的字——都收着呢」。
- [ ] 待拾回超过 10 字时显示 `+N`，点后展开全部区并选中「待拾回」筛选；红框、大号章色角标、灰色小点的三层编码不变。
- [ ] 「导出备份」能调起 iOS 分享/保存面板。
- [ ] 成功保存备份后「我的」显示上次备份时间；取消分享不更新时间。
- [ ] 「恢复备份」能打开 iOS Files 文档选择器，选择 JSON 后确认覆盖并恢复。
- [ ] 清空记录前有确认提示。

## 练习提醒

- [ ] 首个完成日后「我的」页出现一次开启邀请卡；点「先不用」后不再出现，仍可在「练习提醒」设置行手动开。
- [ ] 点「好」弹出系统通知授权；系统层拒绝后设置行显示「需要在系统设置里允许通知」，不反复请求授权。
- [ ] 开启后设置行显示「每天 HH:MM」（习惯练习时刻）；把系统时间调到该时刻能收到通知，文案无动态数字。
- [ ] 当天已盖章后当天不再收到提醒；次日照常。
- [ ] 连续 7 天不打开 App：第 7 条是告别文案，之后静默；重新打开 App 后恢复预约。
- [ ] App 在前台时不弹提醒横幅。
- [ ] 关闭「练习提醒」后不再收到任何拾字通知。
- [ ] 恢复备份到新设备（备份里提醒为开）：不假定已授权，按本机授权状态显示引导或恢复调度。

## 触觉反馈

- [ ] 盖任意一章有一次中等强度触觉；「改一下」回滚成功有一次轻触觉；里程碑庆祝行出现时有一次成功触觉。
- [ ] 点「写好了」进入揭晓、完成普通整组各有一次轻触觉；若整组同时触发里程碑，只出现成功触觉，不叠加轻触觉。
- [ ] 点「笔顺提示」和进入描红各有一次轻微 selection 触觉；描红完成只跟随盖章产生中等触觉，不额外产生揭晓轻触觉。
- [ ] 写字、清空/撤销笔画、双指透视、翻卡和其他普通按钮均无触觉。
- [ ] 连续盖章并点击跳过 1.1 秒停留时，每章各响一次，不丢失、不异常叠加。
- [ ] 系统设置关闭触觉后 App 无任何触觉反馈。

## 布局与输入

- [ ] 深色模式下文字、田字格和按钮可读；首页字块横滑的双侧渐隐在深浅色模式下都自然。
