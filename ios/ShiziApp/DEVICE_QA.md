## 自动真机 Smoke

- [ ] `signing-preflight.sh` 通过，设备状态为 available。
- [ ] 运行 `DEVICE_ID='设备名称或 UUID' SIGNING_XCCONFIG=ios/ShiziApp/Config/Signing.local.xcconfig ios/ShiziApp/scripts/smoke-device.sh`。
- [ ] 普通模式两次 native smoke 均通过，第二次确认 `localStorage` 跨进程重启保留。
- [ ] Smoke 的 `handwritingFlow` 全部通过，确认 pointer 轨迹能落墨且触摸滚动被阻止。
- [ ] Smoke 的软件键盘严格检查通过，`keyboardInsetActive` 为 true，输入框和确认按钮均可达。
- [ ] 加 `DEV_MODE=1` 再跑一次，确认题库质检入口可见且流程通过。
- [ ] 将 `ios/ShiziApp/build/device-smoke/` 下的 JSON 结果随验收记录保留。

## 手写练习

- [ ] 进入写字页后田字格尺寸正常，没有被安全区或底部区域遮挡。
- [ ] 小屏设备上「撤销 / 重写 / 不会写 / 写好了」全部在首屏内，页面不可滚动；深色模式文字、提示笔画和用户墨迹可分辨。
- [ ] 手指在田字格内书写流畅，笔迹不断裂、延迟可接受。
- [ ] 手指写字时页面不滚动、不回弹。
- [ ] 「撤销」按动作倒序回退：写一笔、看一组提示、再写一笔后，连续点三次依次撤回笔画、提示组、笔画；提示组回退瞬时重绘且不闪烁。
- [ ] 普通练习点「重写」会把墨迹和已显示提示全部清空，提示计数回到 0/M；已经用过提示的账不回退，主按钮仍为「写完了」，重看同一组提示不重复记账。
- [ ] 落笔中及抬笔后约 300ms，「不会写 / 写好了」不会被手掌误触；提示动画结束前也不能提交。
- [ ] 「笔顺提示」可以播放，播放后仍能继续书写。
- [ ] 双指或意外三指进入透视后，任一指抬起/取消都立即恢复；余下手指移动不续写，全部离开后下一次落笔正常。
- [ ] 用过一次笔顺提示后只出现一次「双指按住格子，可以透视提示」引导；双指按住时墨迹淡出、提示增强，松开任一指恢复。
- [ ] 看一组或多组提示、只补写剩余笔画后提交：结果页标题为「提交字格」，画面包含仍可见的提示和用户墨迹，自动助手按完整字格给建议。
- [ ] 看多组提示后撤销最后一组：提交快照只含仍可见的提示；全部撤销后完整写对仍自动归类为「看提示写出」，不能当场毕业。
- [ ] 在提示笔画上重复描画不会被机械判为多余笔画；无笔顺数据时自动助手降级但用户仍能判断。
- [ ] 结果页有「写对了 / 写错了」两个明确动作和小号「差一点，记不清」；记不清盖「补拾」、加入本组巩固且不直接毕业。
- [ ] 未用提示且逐笔几何检查明确判差时，点「写对了」只弹一次内联确认；「再对一眼」不记账且同次揭晓不重复弹，「确定写对了」仍按用户选择毕业。
- [ ] 笔画数不同的整体覆盖度检查、自动检查不可用和用过提示的回合均不弹软确认；反馈期点「改一下」后重新选择写对会再次确认。
- [ ] 无辅助写对自动盖「会写」，用过提示写对自动盖「看提示写出」，自由书写选错自动盖「写错了」，点「不会写」自动盖「不会写」。
- [ ] 看过提示后选择写错只增加一条 slow / Again；每个真实回忆回合最多一条 FSRS 事件，不出现 Hard / Easy。
- [ ] 自动盖章后点击盖章、Toast 或结果区会立即换题；连续快速点两次不会跨过下一题，无操作时约 1.4 秒自动换题。
- [ ] 困难字反馈只说已加入本组巩固，不提前显示跨天日期；反馈期「改一下」仍优先回滚而不是换题。
- [ ] 反馈期点「改一下」回到同一份提交字格；记忆、巩固队列、今日不同字/尝试次数和 FSRS 事件全部原子回滚。

## 不会写教学

- [ ] 全新安装进入校准首卡，字与画布渲染完成后「笔顺提示 / 不会写」立即可点；落笔不会重新开始等待，页面明确提示“写不出就点不会写”。
- [ ] 校准首张应为「尴」、第二张应为「嚏」；最后四张不出现明显高于中段的极难字，首次选择写错、记不清、用提示或不会写时只出现一次“忘了正常”宽慰。
- [ ] 点「不会写」后立即进入「1/2 描写」，不先出现结果、盖章、等待动画或确认弹层；同一个字再次求助也完整进入两步教学。
- [ ] 全 App 首次说明直接显示在描写页内，画布同时可写；第一笔有效描迹后自然收起并记为已看，未落笔返回或强杀后仍会显示。
- [ ] 第 1 步轮廓可见，只保留「撤销 / 重描 / 描好了，自己写」；主按钮初始不可用，实际落笔后才可用。
- [ ] 进入第 2 步后轮廓、fallback 字形、描写墨迹和提示全部消失，只保留「撤销 / 重写 / 再描一遍 / 写好了」。
- [ ] 第 2 步选择写错会回到第 1 步；选择写对只完成教学检查，不写 Good，仍需本组稍后无辅助写对。
- [ ] 第 1 步、第 2 步和教学完成反馈期间退出再进入，都恢复同一目标字、墨迹与教学阶段。

## 本组巩固与返回

- [ ] 看提示写出、写错、不会写都会在本组稍后重现，默认尽量隔两个其他回合，不等待 6/12 小时。
- [ ] 分别让困难字出现在第 1、14、15 位：第 1 位尽量隔两题；第 14/15 位不足间隔时可回转，不提前结算也不卡死。
- [ ] 同一个字连续失败至少 10 次、或组里只剩一个困难字时仍可继续；今日不同字只加 1，尝试次数逐次增加。
- [ ] 普通组达到 20 次尝试或约 6 分钟时，只在换题间隙出现一次节奏提醒；选择「继续练」后本组不再重复出现，首次校准不会被截断。
- [ ] 节奏提醒选择「明天接着拾」后，未解决字按原队列顺序出现在下一组最前面；已覆盖完整基础组时今日完成态成立，未覆盖的短组不误记完成。
- [ ] 再次提示、写错或不会写继续留在队列；只有全新回合无提示、无描写并确认写对才毕业。
- [ ] 新组首张显示「已练 0/15」；首次提交后显示「已练 1/15」；同字巩固、描写和教学第 2 步不重复增加已练数。
- [ ] 目标未全部覆盖时显示「已练 X/Y · 待巩固 N」；全部覆盖但仍未毕业时只显示「待巩固 N」，同字多个待办只计 1。
- [ ] 320 / 375 / 390 / 430pt、动态字体、Y > 15、N 为两位数时顶栏不换行，不与返回或「＋加字」重叠；不存在点阵或图形进度条。
- [ ] 结算页每个基础字只出现一次，保留首次困难标签，并为已毕业困难字显示「已独立」。
- [ ] 「不想练」会从当前待巩固中排除并记为隐藏，不伪装成独立写出，也不永久卡住本组。
- [ ] new / review / focus / calibrate 四种入口都遵守同一完成条件。
- [ ] 顶部显示标准返回符号，点击区至少 44 × 44pt；点击返回与左缘侧滑都直接保存并回首页，不出现退出确认、“进度已保存”Toast，或旧主页 / 练习页闪帧。
- [ ] 加字面板打开时，第一次返回只关闭面板；再次返回才离开练习。首页左缘侧滑无练习退出行为。
- [ ] 连续执行至少 3 次“进入未完成组 → 左缘侧滑 → 首页 → 续练”；每次恢复同一字、阶段、墨迹、提示、撤销状态和队列，不前跳、不重复记账、不堆 history。
- [ ] 模拟存储空间不足或写入失败时，返回留在当前练习并提示重试；恢复写入后才能回首页。

## 本地数据

- [ ] 完成至少一题后退出并重启 App，已拾字/回炉/复习状态仍存在。
- [ ] 未完成组退出后首页显示续练态；重启后恢复 recall、tracing、postTraceRecall、revealDecision、reinforcement 五种状态中的原状态。
- [ ] 在每种状态分别杀掉 App 再启动，墨迹、可见提示、提交快照、教学阶段和队列不丢、不重复记账。
- [ ] 将未完成组跨过本地午夜再打开，仍优先继续原组；不同字数不重复增加，昨天的组不被错误结算。
- [ ] 完成第一组后首页固定显示「今日已拾 N 个字」；到期不足 15 字的短组做完也算完成，加练或新增到期字不会把完成态改回未完成。
- [ ] 快速连点盖章、重写等控件不会触发页面双击缩放，双指捏合缩放仍可用。
- [ ] 完成后加练一组，今日计数继续累加；中途退出显示「继续加练这组」。
- [ ] 首页「拾字第 N 天」按累计练习日显示：当天完整做完第一组才 +1，只盖几章或只打开不计；与「我的」页「练习天数」一致，且同日加练不重复计。
- [ ] 老数据升级后，累计天数 = 升级前的历史打开天数 + 升级后的完成日数，不清零。
- [ ] 首个完成日结算页出现「见字如晤」庆祝行，且只出现这一次。
- [ ] 困难字最终独立毕业后才显示 FSRS `dueDay`，且最早是本地次日；导出导入后同一时刻的状态和日期一致。
- [ ] 构造或恢复大量到期字时，当天最多进入 30 个（两组）；完成后首页提供新字入口，不显示未封顶的积压总数。
- [ ] 字盒仍可查看完整到期数；关闭重开或恢复备份后，当天已占用的复习预算不会重新放量。
- [ ] 15 字满组添加本组外新字后分母变 16，当前完整回合结束后的下一题就是该字，原 15 字均保留；继续添加可增长到 17、18。
- [ ] 添加本组尚未出现、已练过、当前正在练、已待巩固或已手动排队的字时，只移动或创建一次复写，不增加分母、不制造重复机会。
- [ ] 最后一张基础题进入反馈后加入当前字或已练字，立即返回首页再续练：不提前结算，先恢复反馈，再把该字作为下一题复写。
- [ ] 输入「甲乙甲」只按「甲、乙」出现；多字、多次确认保持 FIFO，优先于已 ready 的巩固字；Toast 描述真实顺序。
- [ ] 普通回忆在结果自评后切到新增字；「不会写」必须完成完整两步教学和结果自评后再切。返回、强杀和跨午夜后顺序不丢。
- [ ] new / review / focus / calibrate 都遵守下一题契约；校准新增字参与学习，但原 15 个摸底样本不被替换，额外题结果不进入校准统计或改变出题偏好。
- [ ] 首页有今日记录时显示「今日拾得」并包含四类结果；超过 4 个字可顺滑横滑浏览，左右边缘渐隐且后续字自然半露；无今日记录时回退「昨日拾得/上回拾得」，不会写的字为克制灰态。
- [ ] 字盒打开后先看到统计和最多两行「待拾回」，「全部 N 字」默认折叠，「＋加字」在折叠态仍可见；待拾回为空时显示「没有待拾回的字——都收着呢」。
- [ ] 待拾回超过 10 字时显示 `+N`，点后展开全部区并选中「待拾回」筛选；红框、大号章色角标、灰色小点的三层编码不变。
- [ ] 「导出备份」能调起 iOS 分享/保存面板。
- [ ] 成功保存备份后「我的」显示上次备份时间；取消分享不更新时间。
- [ ] 「恢复备份」能打开 iOS Files 文档选择器，选择 JSON 后确认覆盖并恢复。
- [ ] 恢复确认同时显示“当前 N 字（最后练习日期）→ 备份 M 字（备份日期）”，取消时当前数据完全不变。
- [ ] 备份包含 session v2、FSRS review log 和描写教学标记；换机恢复后未完成组可继续。
- [ ] 首启页常驻「之前练过？恢复一份备份」，选择合法备份后进入老用户首页。
- [ ] 完成第 5 个练习日或记录 30 字、且 7 天未备份时，「我的」出现备份提醒；第 3 个完成日结算页仅出现一次轻提示。
- [ ] 清空记录前有确认提示；恢复和清空后出现一次「撤销恢复/撤销清空」，点击后完整还原操作前数据。
- [ ] 连续执行两次破坏性操作，撤销只恢复第二次操作前的数据；导出的 JSON 不含 `shizi.safety.v1`。

## 练习提醒

- [ ] 完成首次校准后，结算页就地显示练习提醒邀请且默认关闭；点开启会触发系统通知授权，拒绝后仍可正常点“明天见”或继续练。
- [ ] 首个完成日后「我的」页出现一次开启邀请卡；点「先不用」后不再出现，仍可在「练习提醒」设置行手动开。
- [ ] 点「好」弹出系统通知授权；系统层拒绝后设置行显示「需要在系统设置里允许通知」，不反复请求授权。
- [ ] 开启后设置行显示「每天 HH:MM」（习惯练习时刻）；把系统时间调到该时刻能收到通知，文案无动态数字。
- [ ] 当天已盖章后当天不再收到提醒；次日照常。
- [ ] 连续 7 天不打开 App：第 7 条是告别文案，之后静默；重新打开 App 后恢复预约。
- [ ] App 在前台时不弹提醒横幅。
- [ ] 关闭「练习提醒」后不再收到任何拾字通知。
- [ ] 恢复备份到新设备（备份里提醒为开）：不假定已授权，按本机授权状态显示引导或恢复调度。

## 触觉反馈

- [ ] 盖任意一章有一次中等强度触觉；「改一下」回滚成功有一次轻触觉；里程碑庆祝行出现时有一次成功触觉。
- [ ] 点「写好了」进入揭晓、完成普通整组各有一次轻触觉；若整组同时触发里程碑，只出现成功触觉，不叠加轻触觉。
- [ ] 点「笔顺提示」有一次 selection；点「不会写」立即进入描写且只产生一次 selection，不先产生盖章触觉。第 1 步描完切到第 2 步时无触觉；第 2 步点「写好了」只有一次揭晓轻触觉，确认写对不再叠加盖章触觉。
- [ ] 写字、清空/撤销笔画、双指透视、翻卡和其他普通按钮均无触觉。
- [ ] 连续自动盖章并等待 1.4 秒反馈时，每章各响一次，不丢失、不异常叠加。
- [ ] 系统设置关闭触觉后 App 无任何触觉反馈。
